## 演習 1: 思考と最終回答を分離して色分け表示

```python
import re
from colorama import Fore, init
init()

def pretty(text: str) -> None:
    m = re.search(r"<think>(.*?)</think>(.*)", text, re.DOTALL)
    if m:
        print(Fore.CYAN + "[THINK]\n" + m.group(1))
        print(Fore.GREEN + "[ANSWER]\n" + m.group(2).strip())
    else:
        print(text)
```

`colorama` が無ければ ANSI エスケープでも可。

## 演習 2: 4 種混合データで学習

加減乗除をランダム生成するヘルパを作る:

```python
import random, operator
ops = [(operator.add,"+"), (operator.sub,"-"),
       (operator.mul,"*"), (operator.floordiv,"/")]
def gen():
    op, sym = random.choice(ops)
    a, b = random.randint(2, 30), random.randint(2, 10)
    return f"{a} {sym} {b} はいくつ？", op(a, b)
```

500 件でミニ SFT したあと、4 演算の内訳別正答率を見ると、
**除算だけ正答率が低い** ことが多い（余りや分数の扱い）。

## 演習 3: DeepSpeed ZeRO レベル比較

| Stage | パラメータ | 勾配 | オプティマイザ状態 | CPU offload |
|---|---|---|---|---|
| ZeRO-0 | 全保持 | 全保持 | 全保持 | なし（= DDP） |
| ZeRO-1 | 全保持 | 全保持 | **分散** | 可 |
| ZeRO-2 | 全保持 | **分散** | **分散** | 可 |
| ZeRO-3 | **分散** | **分散** | **分散** | 可（最大削減） |

7B モデルを 8×H100 で学習するには最低でも ZeRO-2、長文SFTや 32B以上では ZeRO-3 が実質必須。
その代わり通信量が増えるので、**高速な NVLink / Infiniband** が望ましい。
