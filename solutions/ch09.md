## 演習 1: Mixture-of-Thoughts を覗く

```python
from datasets import load_dataset
ds = load_dataset("open-r1/Mixture-of-Thoughts", split="train", streaming=True)
first = next(iter(ds.take(100)))
print(first.keys())    # => dict_keys(['messages', 'domain', ...])
```

平均トークン長は `tokenizer` を通して

```python
lens = []
for row in ds.take(100):
    lens.append(sum(len(t) for t in [m["content"] for m in row["messages"]]))
print(sum(lens)/len(lens))
```

で調べられる。多くの CoT サンプルは数千〜数万文字に達する。

## 演習 3: 蒸留 vs GRPO の実験設計

**仮説**: 小型モデルには RL より蒸留の方が効果が高い。

| 項目 | 設定 |
|---|---|
| ベースモデル | Qwen2.5-0.5B |
| タスク | GSM8K（算数文章題） |
| Arm A: 蒸留 | `Mixture-of-Thoughts` の数学サブセット 10K 件で 1 epoch SFT |
| Arm B: GRPO | 同じ問題プールで `accuracy_reward` のみで 2000 step |
| Arm C: 両方 | A で蒸留 → B で RL |
| 評価 | GSM8K test の pass@1, pass@8 |
| 期待結果 | A > B（小型では RL 単独は推論能力を伸ばしきれない） |

論文の結論と合致するはず — もし違ったら、実装バグ or ハイパラを見直す良い機会。
