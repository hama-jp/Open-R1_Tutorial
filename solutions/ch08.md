## 演習 2: `<think>` 2 回ブロックも検出する正規表現

```python
FORMAT_RE = re.compile(
    r"(<think>.*?</think>\s*){1,2}<answer>(-?\d+)</answer>", re.DOTALL
```

## 演習 3: `cosine_reward` が短い CoT を奨励する理由

Open-R1 の `cosine_reward` は概ね次のような形:

```python
def cosine_reward(completion_len, max_len, correct: bool):
    if correct:
        return 0.5 + 0.5 * math.cos(math.pi * completion_len / max_len)
    else:
        return -0.5 + 0.5 * math.cos(math.pi * completion_len / max_len)
```

- `completion_len=0` → `cos(0)=1` → 正解なら +1、不正解なら 0
- `completion_len=max_len` → `cos(π)=-1` → 正解なら 0、不正解なら -1

つまり

- **正解したときは短いほど高報酬**（無駄に長い思考は減点）
- **不正解のときは長いほど減点が軽い**（探索を続ける余地を残す）

という方針を 1 本の式で表現している。
