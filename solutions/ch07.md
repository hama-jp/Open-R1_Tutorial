# 第7章 解答

## 演習 1: PPO vs GRPO の行数差

| 削除できる主要コード | PPO → GRPO |
|---|---|
| 価値関数ネットワーク `V_\phi` の forward/loss | **不要** |
| GAE の計算（ループで $\delta$ を累積） | **不要** |
| `value_loss = F.mse_loss(v_pred, returns)` | **不要** |
| アドバンテージ正規化 | `(r - mean)/std` でシンプルに置換 |

体感としては **価値関数関連のコードが 30〜50% 消える**。

## 演習 2: `GRPOTrainer` のグループ内標準化を見つける

`trl/trainer/grpo_trainer.py` の `_generate_and_score_completions` 付近に、

```python
rewards = ...                                   # (B*G,)
rewards = rewards.view(B, G)
advantages = (rewards - rewards.mean(dim=1, keepdim=True)) / (rewards.std(dim=1, keepdim=True) + 1e-8)
advantages = advantages.view(-1)
```

のような行がある（バージョンで微妙に変化）。ここが本章 7.2 の式にそのまま対応。

## 演習 3: 応答長報酬での GRPO

[`examples/ch07/grpo_length.py`](../examples/ch07/grpo_length.py) を実行。
学習初期は 200〜300 文字や 30 文字の応答が混在するが、
数ステップで **100 文字近辺にピーク** するヒストグラムに寄っていくのが観察できる。

TensorBoard で `reward/mean` を見ると単調に上昇するはず。
逆に `reward_std` が 0 に張り付いたら、全応答が同じ長さになってしまい、
GRPO のアドバンテージが 0 近くに落ちる（学習停止の危険）。
