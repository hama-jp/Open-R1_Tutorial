## 演習 1: 学習指標と "aha moment"

TensorBoard で **`reward/eval_reward`** と **`completion_length`** を同時にプロット。
典型的な進行:

| Step | reward | length | 観察 |
|---|---|---|---|
| 0 | 0.05 | 120 | ランダム式が出がち |
| 50 | 0.15 | 210 | 書式報酬に適応しはじめ `<think>` が出る |
| 150 | 0.35 | 400 | 式は数字を全部使うように |
| 300 | 0.60 | 550 | 自己修正フレーズの登場（"Wait..."）|
| 500 | 0.70 | 600 | プラトー |

"aha moment" は **reward が急に伸び、同時に length も伸びる** タイミングに対応することが多い。

## 演習 2: ソフト報酬との比較

[`examples/ch12/soft_reward.py`](../examples/ch12/soft_reward.py) で定義した
`1/(1+|v-t|)` 版に差し替えて比較すると、

- ソフト: 序盤の勾配が流れやすく、**最初の 100 step で大きく伸びる**
- 二値: 序盤は 0 ばかりでほぼ静止、ある閾値を超えると急に伸びる

最終到達性能はタスク次第だが、**"exploration が難しい問題ではソフト報酬が有利"** という
一般的な観察と一致する。

## 演習 3: GSM8K でのミニ実験

```python
import re
def gsm8k_reward(completions, answer=None, **_):
    def parse(a): 
        return a.split("####")[-1].strip().replace(",", "")
    out = []
    for c, gold in zip(completions, answer):
        m = re.search(r"####\s*(-?\d+\.?\d*)", c) or re.search(r"\\boxed\{(-?\d+\.?\d*)\}", c)
        out.append(1.0 if (m and parse(gold) == m.group(1)) else 0.0)
    return out
```

データ 10 問だけでは GRPO が安定しにくい。最低 500 問程度で動かすと、
Qwen2.5-1.5B で **10〜20 ポイント程度の改善** が 500 step 内で観測できる。
