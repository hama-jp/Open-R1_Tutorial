## 11.5 動作確認: まずは推論

DeepSeek-R1-Distill-Qwen-1.5B は 3GB 程度で、多くの環境で動きます。

```python
# examples/ch11/quick_infer.py
from transformers import AutoTokenizer, AutoModelForCausalLM
import torch

mid = "deepseek-ai/DeepSeek-R1-Distill-Qwen-1.5B"
tok = AutoTokenizer.from_pretrained(mid)
model = AutoModelForCausalLM.from_pretrained(
    mid, torch_dtype=torch.bfloat16, device_map="auto"
)

prompt = tok.apply_chat_template(
    [{"role": "user", "content": "1/3 + 1/6 を計算せよ。"}],
    tokenize=False, add_generation_prompt=True,
)
inputs = tok(prompt, return_tensors="pt").to(model.device)
out = model.generate(**inputs, max_new_tokens=1024, temperature=0.6, top_p=0.95)
print(tok.decode(out[0], skip_special_tokens=True))
```

`<think>` で始まる思考過程と、その後の最終回答 `1/2` が得られれば成功です。

> ✅ **成功時の期待出力（抜粋）**
> ```
> <think>
> 1/3 と 1/6 の最小公倍数を取る。6 で揃えると 2/6 + 1/6 = 3/6 = 1/2。
> </think>
> 答えは 1/2 です。
> ```
>
> ❌ **よくある失敗ログと対処**
> - `torch.cuda.OutOfMemoryError: CUDA out of memory` → T4 (15GB) で動かすなら `torch_dtype=torch.float16` に変更
> - `ValueError: ... unrecognized arguments: temperature` → `do_sample=True` を追加
> - 出力が空 / `<think>` が閉じない → `max_new_tokens` を増やす（1024→4096）

## 11.6 ミニ SFT 実験：小さな算数データで試す

### 11.6.1 データセット作成

まず小さな算数データを自作します。

```python
# examples/ch11/make_tiny_ds.py
import json, random, pathlib
random.seed(0)

rows = []
for _ in range(200):
    a, b = random.randint(1, 50), random.randint(1, 50)
    rows.append({
        "messages": [
            {"role": "user", "content": f"{a} + {b} はいくつですか？"},
            {"role": "assistant",
             "content": f"<think>{a}と{b}を足すと{a+b}。</think>答えは{a+b}です。"},
        ]
    })

pathlib.Path("tiny_addition.jsonl").write_text(
    "\n".join(json.dumps(r, ensure_ascii=False) for r in rows)
)
```

### 11.6.2 SFT スクリプト

```python
# examples/ch11/tiny_sft.py
from trl import SFTTrainer, SFTConfig
from transformers import AutoTokenizer, AutoModelForCausalLM
from datasets import load_dataset
import torch

model_id = "Qwen/Qwen2.5-0.5B-Instruct"
tok = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(model_id, torch_dtype=torch.bfloat16)

ds = load_dataset("json", data_files="tiny_addition.jsonl", split="train")

config = SFTConfig(
    output_dir="out-sft",
    num_train_epochs=1,
    learning_rate=2e-5,
    per_device_train_batch_size=2,
    gradient_accumulation_steps=4,
    bf16=True,
    logging_steps=10,
    max_seq_length=512,
    packing=False,
)

trainer = SFTTrainer(
    model=model,
    args=config,
    tokenizer=tok,
    train_dataset=ds,
)
trainer.train()
trainer.save_model("out-sft/final")
```

T4 GPU（15GB）でも 5〜10 分で終わります。

### 11.6.3 学習前後の比較

```python
# examples/ch11/compare.py
from transformers import AutoTokenizer, AutoModelForCausalLM
import torch

def ask(path, q):
    tok = AutoTokenizer.from_pretrained(path)
    m = AutoModelForCausalLM.from_pretrained(path, torch_dtype=torch.bfloat16).to("cuda")
    prompt = tok.apply_chat_template(
        [{"role": "user", "content": q}], tokenize=False, add_generation_prompt=True
    )
    ids = tok(prompt, return_tensors="pt").to("cuda")
    out = m.generate(**ids, max_new_tokens=200, temperature=0.0)
    return tok.decode(out[0], skip_special_tokens=True)

q = "17 + 28 はいくつ？"
print("BEFORE:", ask("Qwen/Qwen2.5-0.5B-Instruct", q))
print("AFTER :", ask("out-sft/final",            q))
```

SFT 後は `<think>` タグを使って丁寧に足し算を解くようになるはずです。

## 11.7 よくあるエラーと対処

| エラー | 原因 | 対処 |
|---|---|---|
| `torch.cuda.OutOfMemoryError` | バッチ・系列長が大きすぎ | `per_device_train_batch_size=1`, `gradient_accumulation_steps`↑ |
| `bf16 not supported` | Ampere 未満（T4 等） | `fp16=True` に変更 |
| `No module named 'flash_attn'` | FlashAttention 未導入 | `attn_implementation="sdpa"` に変更 |
| `Transformers version mismatch` | trl / transformers 不整合 | `uv pip sync` で再インストール |
| `HFValidationError: ... gated` | アクセス許可が必要なモデル | HF 上で同意してから再試行 |

## 11.8 DeepSpeed ZeRO-3 を使う

本番の 7B 学習には **DeepSpeed ZeRO-3** が必須です。Open-R1 は設定ファイルを同梱しています。

```bash
$ accelerate launch \
    --config_file configs/zero3.yaml \
    src/open_r1/sft.py --config recipes/OpenR1-Distill-7B/sft/config.yaml
```

ZeRO-3 は **オプティマイザ状態・勾配・モデルパラメータ** を全 GPU に分散して持たせる方式で、
8× H100 80GB で 7B モデルを学習できるのはこの仕組みのおかげです。
